This projects collects all published (aka tagged) Symfony Standard Distribution releases to ease the author's update workflow.

"Official" update workflow

After the installation of the setup, you usually creates a clean git repository over the latest version of Symfony Standard Distribution and do no track changes (Source control on a composer-created framework project).

The "official" update process involves download the new composer or deps related files and run the suitable vendor manager to obtain new versions (see Symfony 2.1.3 released). This works but some files are not updated. In almost all cases this is not really a issue.

"Alternative" update workflow

The alternative uses merge of latest versions (provides by this repository as a remote) into the master branch of the code repository. This merges provides updated versions of composer and deps files and all related stuff of the distribution.

Initial setup

When starting a new project:

  1. Add this repository as remote git remote add symfony-standard http://github.com/ocubom/symfony-standard-releases.git
  2. Create your master branch from one version git checkout -b master symfony-standard/2.1 # or 2.0
  3. Install vendors composer install

The workflow can be also used on an existing project. In this case, the first setup is a bit more complex as it requires an initial merge.

  1. Add this repository as remote git remote add symfony-standard http://github.com/ocubom/symfony-standard-releases.git
  2. Merge the version you prefer into your master branch.

Upgrade

After the initial setup, the upgrade will be fast:

  1. Get latest changes in remote repository: git remote update
  2. Merge latest version into your master branch: git merge symfony-standard/2.1 # or 2.0. This may generate some conflicts that you must solve by hand.

Note

This repository contains just clones of the original commits tagged on the original Symfony Standard Distribution repository. Original author and dates are maintained.
